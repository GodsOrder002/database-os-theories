## 1) 你当前字段分层，其实已经是“纯推断链 + 混合编译链”

你说的 B/C 可以直接压成两条链：

### 纯推断链（不与我方目标混合）

* `collect_fakes`：证据快照（你从 `fakes_cards` 拉来的那一坨）
* `his_hunts`：推断依据组合（你挑出来作为推断输入的那部分/那组）
* `his_results`：推断结果（对目标组合的预测/推断输出）

**边界句（纯推断链宪法）**：

1. `his_hunts ⊆ collect_fakes`（依据必须来自证据快照；不能凭空长出来）
2. `his_results` 只能由 `{collect_fakes, his_hunts}` 推出，**不得引用 `my_hunts`**（否则你就混合了）
3. `his_results` 永远是模型态，不自动升级为事实态（它只能影响后续混合/权衡）

### 混合编译链（把“我想猎杀什么”拉进来）

* `my_hunts`：我方猎杀依据（你说它也是从 `fakes_cards` 拉，只是过滤 `player_id=我`）
* `mix_to_hunts`：动态分析体（你说得很准确：`my_hunts + collect_fakes + his_*` 的混合态）
* `my_results`：我最终猎杀的组合（只在混合之后产生）

**边界句（混合链宪法）**：
4) `mix_to_hunts` 必须显式声明它引用了哪些输入块（至少能追溯到：`my_hunts / collect_fakes / his_results`）
5) `my_results` 只能从 `mix_to_hunts` 产生；**不能直接从 collect_fakes 或 his_results “跳级”写结论**
6) 混合链不得倒灌：`mix_to_hunts / my_results` 不得回写改造 `collect_fakes` 与 `his_*`（否则证据就被决策污染）
